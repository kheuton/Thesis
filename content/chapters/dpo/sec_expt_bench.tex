We replicate the 7 primary tasks from \citet{gengBenchmarkingPtOPnO2024}'s end-to-end prediction and optimization (PnO) benchmark, using their implementation (when possible) and dataset settings. We omit their novel advertising task, as its online nature precludes many common decision-aware methods. See App.~\ref{app:bench} for task specific descriptions.

We add a Differentiable Perturbed Optimizer method \cite{berthetLearningDifferentiablePerturbed2020}, which was referred to in \citet{gengBenchmarkingPtOPnO2024} but not actually benchmarked. While the benchmark authors provide some code for DPO, its implementation was incomplete so we use instead code from  \citet{TueroPytorchDPO}. While \citeauthor{gengBenchmarkingPtOPnO2024}'s provided software contains some DPO code, it is not fully implemented.

While the original benchmark paper performed minor hyperparameter search over learning rate alone, we hypothesize that more thorough hyperparameter searching will have an impact on which models are most successful. We conduct two stages of hyperparameter search. In the first we search an entire grid of:
\begin{enumerate}
    \item Adam Learning Rate: $[0.01, 0.005, 0.001]$. The original paper explored $[0.1, 0.05]$ as well, but these were omitted to save time as many models seemed to prefer $0.005$ over $0.01$.
    \item Batch size. The original paper used either full-batch gradient descent, or for a few methods \TODO{Fill}, a batch-size of 1. Here we explore a minibatch size of 32 (4 for bipartite matching with only 20 training examples) in addition to the default full-batch or single-instance behavior.
\end{enumerate}

After an initial search of this Learning Rate $\times$ Batch Size $\times$ Tasks $\times$ Methods $= 3 \times 2 \times 7 \times 12 = 504$ training runs, we select the best learning rate and batch size for each Task/Method pair based on validation decision regret. Then, we conduct a second hyperparameter sweep of any method-specific hyper parameters. Although this may "unfairly advantage" any method with additional hyperparameters as it will receive more trials, in this work we are not necessarily interested in determining the optimal method, but instead with assessing what is possible on these benchmarking tasks.

\subsection{Results on Tasks Where Decision-unaware Methods are Successful}
We report the results of this exercise for the Knapsack (real and synthetic), Scheduling, Top-K, and Bipartite Matching tasks in Figure \ref{fig:bench_bump}. We group these 5 tasks together because of a surprising finding: a two-stage decision-blind approach is superior to nearly all methods, save for listwise-LTR, with which it has similar performance. 

We hypothesize the surprisingly strong performance of a decision-unaware approach is due to these problems being well-specified. The MLP model with a single 32-unit hidden layer used by the benchmark has sufficient capacity to represent the relatively simple data generating processes in this task. Indeed in the Top-K (Cubic) task, we see that some methods are capable of virtually 0 prediction error on heldout data. The knapsack and scheduling tasks show a similar scenario: regret is between $1\%-5\%$.

The bipartite matching task may be an exception. We see that LTR methods are generally capable of approximately $90\%$ regret, while the decision-blind approach is near $92\%$. Given the large gap between the predicted decisions and optimal, this task may be too difficult for any extant method with only 20 training examples.

\subsection{Results on Budget Allocation Task}
Results on the Budget Allocation task are shown in Figure \ref{fig:budget_bump}. Here we see an opposite story: the decision-blind approach is worse than all but one decision-aware method. The two-stage model only achieves $35\%$ regret, while the best decision-aware methods have less than $5\%$ regret on heldout data. We argue this task is much more informative for decision-aware model selection than the rest, because it is the only task where decision-aware models have clear advantages over decision-blind approaches. We hypothesize that this is due to the complex data-generating process: this is real world data on the click-through rates of different websites. Additionally, the cost function is markedly different from prediction error. Here we are interested in the submodular probability of a user clicking at least one advertisement.

\subsection{The Portfolio Task}
Results on the Portfolio task are presented in Figure \ref{fig:portfolio_bump}. These are presented separately because we believe this task should not be included in benchmarks of decision-aware methods without sufficiently noting that this is not a combinatorial optimization task. This task is to optimize the allocation of funds in an investment portfolio, and the allocation fractions are treated as continuous random variables. Accordingly, the objective is not piecewise constant with respect to model parameters and has informative gradients.

When originally used as a decision-aware benchmarking task in \citet{shahDecisionFocusedLearningDifferentiable2022}, it was proposed as a "stress-test," in that it was a task where decision-aware methods were known to preform poorly. Here \citeauthor{shahDecisionFocusedLearningDifferentiable2022} show how their novel decision-aware method (LODL) performs at the same level as the decision-blind two stage model. Furthermore, these authors show that the best method in this case is simply optimizing the objective directly. Here, this is called Decision Focused Learning (DFL). While \citeauthor{gengBenchmarkingPtOPnO2024} implement a DFL method in the benchmark, it \textit{always} uses a surrogate loss, even though this is not necessary on the Portfolio task. We remedy this, and highlight the strong performance of DFL on this task in Figure \ref{fig:portfolio_bump}.
